=========================================================================
 Beamline DAC Data Tool  (rename pending: SQUISHE or CULET)
 Release notes, version 1.4.0-preview  (build date 2026-07-11)
 Supersedes v1.3.0 (released 2026-06-16)
 NSLS-II beamline 22-IR-1, Lee Lab
=========================================================================

SCOPE
  Single-window Windows application for the reduction and publication-
  quality visualization of visible / near-IR optical-absorption spectra
  from diamond-anvil-cell experiments: segment concatenation, absorbance
  computation A = -log10[(S - D)/(B - D)], FFT-notch defringing, Savitzky-
  Golay smoothing (Igor-pipeline equivalent), 2D overlay / 2D stacked /
  3D ridge plotting, and CSV + figure export with full provenance.

-------------------------------------------------------------------------
1. DATA INGESTION
-------------------------------------------------------------------------
1.1  Configurable filename grammar ("Name format", left panel).
     The fixed convention vis_{DAC}_{Sample}[_{P}][_bg|_s][_C|_D][_n]
     [.{seq}] is no longer a requirement. A grammar editor lets the user
     declare, per profile: the token separator, an optional fixed prefix,
     the segment-number separator, the pressure decimal character
     ('p', '.', or ','), unit suffixes to strip (e.g. gpa, kbar), and
     keyword sets for the background, sample, dark, compression, and
     decompression tokens.
1.2  Teach-by-example. The user selects one real filename from the input
     folder; it is split into pieces, and each piece is labeled from a
     dropdown (DAC / sample / pressure / role / branch / replicate /
     ignore). The grammar is inferred from these labels.
1.3  Live validation. Every file in the folder is parsed continuously
     against the working grammar and shown in a preview table with the
     extracted fields; rows are green (parsed), red (skipped, with the
     specific reason), or blue (manually fixed). A running "matched
     N / M files" counter is displayed.
1.4  Per-file overrides. Any file can be corrected by hand (all fields),
     resurrected when no grammar can parse it, or excluded from the run.
     Overrides are stored per input folder and persist across restarts.
1.5  Named profiles. Grammars are saved under user-chosen names and
     selected per run; the active profile is shown on the main window.
     The classic 22-IR-1 grammar remains the built-in default, is
     byte-identical to previous releases, and requires no setup.
1.6  Guidance. The editor contains a plain-language guide sidebar with a
     worked example, ghost example text inside every input box, and
     hover help on every control. Skipped files prompt a hint in the
     run log. The parser layer is covered by 10 dedicated unit tests.

-------------------------------------------------------------------------
2. SESSION MANAGEMENT
-------------------------------------------------------------------------
2.1  Multi-tab sessions (browser style). Each tab holds an independent
     dataset with its own folders, every plot setting, trace visibility,
     caches, and undo/redo history. Tabs are created with "+" or Ctrl+T,
     renamed by double-click, closed by Ctrl+W / middle-click / the
     close glyph, and cycled with Ctrl+Tab. Tabs auto-name from the data
     folder; the window title follows the active tab.
2.2  Recent runs are listed on every blank tab for one-click reload.
     "Load previous run" reopens any processed output folder in viewer
     mode without recomputation.

-------------------------------------------------------------------------
3. DATA INSPECTION
-------------------------------------------------------------------------
3.1  Raw data table. A resizable drawer beneath the plot (Ctrl+D or the
     "Data table" button) shows, for the selected trace: wavelength (nm),
     wavenumber (cm^-1), absorbance, and the raw dark / background /
     sample counts, plus defringed and smoothed columns when enabled.
3.2  Export paths: copy the table as TSV to the clipboard, or open it
     directly in Excel.

-------------------------------------------------------------------------
4. VISUALIZATION AND FIGURE PREPARATION
-------------------------------------------------------------------------
4.1  Journal presets. One selection applies a publisher's current figure
     specification: column width, typeface class, final font sizes, line
     weight, spine/tick style, and 300 dpi. Included: Nature (89 /
     183 mm, sans-serif), Science (5.7 / 12.1 / 18.4 cm, sans-serif),
     RSI / AIP (3.37 / 6.69 in), APS (3.4 / 7.0 in, serif), Elsevier
     (90 / 190 mm).
4.2  WYSIWYG export preview. The on-screen figure can be rendered at the
     exact export dimensions so proportions and printed text sizes are
     judged before saving. Export formats: PNG / PDF / SVG / EPS / TIFF.
4.3  Legend improvements. Legend keys are thick color blocks by default
     (matching the 3D legend; a line style remains selectable). For
     dense pressure series an optional mode replaces the legend with a
     continuous pressure colorbar automatically when more than ten
     traces share a continuous colormap.
4.4  Axis control. One consistent dropdown per axis: X unit (wavelength
     nm / wavenumber cm^-1 / photon energy eV), Y data (absorbance or a
     raw channel), mirrored top axis in a second unit, and a right axis
     (mirror or percent transmittance). Additional controls: Flip X,
     Flip Y, axis line thickness (pt), and axis-label gap (pt, per axis).
4.5  2D view control. A pan pad with press-and-hold repeat (5% of the
     span per step), zoom in/out buttons (10% per step) scoped to X, Y,
     or both, and Fit / Fit X / Fit Y. Keyboard: arrow keys pan, + and -
     zoom, 0 fits; all ignored while typing in a field. Drag-box zoom
     and scroll-at-cursor zoom are unchanged.
4.6  3D view control. The same keys drive the 3D camera in ridge mode:
     arrows orbit in 3-degree steps (azimuth wraps at +/-180, elevation
     clamped 0-90), + / - zoom within 0.5-2.0x, 0 resets; the on-panel
     sliders track the keyboard. Slider and preset controls unchanged.
4.7  3D Z range correction. The automatic Z range now spans the full
     data range. The previous behavior (silent truncation at the 99th
     percentile, which could hide real absorbance maxima) is now an
     explicit "Clip Z spikes" option, default off, and is reported in
     the run log when active.

-------------------------------------------------------------------------
5. TRACEABILITY
-------------------------------------------------------------------------
5.1  Provenance sidecars. Every reduction run and every export (figures
     and CSVs) writes a JSON sidecar recording the tool version,
     processing parameters, timestamps, and SHA-1 hashes of the files
     written, sufficient to reconstruct any figure or table.
5.2  The run log records axis changes, preset applications, naming-
     profile selection, and skipped-file counts; it can be copied or
     exported as a methods-section text block.

-------------------------------------------------------------------------
6. INTERFACE
-------------------------------------------------------------------------
6.1  Typography and layout normalization; section headers standardized;
     settings grouped in five tabs (Plot / Axes / Style / Data / Export)
     with a search field ("Find a setting") that filters sections.
6.2  Hover help on every control, governed by a single Helper switch;
     F1 shows the shortcut list; the left "Guide / notes" panel contains
     a quick start, a panel-by-panel guide, an absorbance reference, and
     a persistent free-text notes page with adjustable font.
6.3  Confirmation toasts for clipboard and save actions; a prominent
     status line (active tab, plot mode, preset, traces shown); themes
     including a true-black default; scroll-wheel input no longer
     changes control values (panel scrolling only).
6.4  About dialog with version, credits, and repository link.

-------------------------------------------------------------------------
7. PACKAGING AND INTEGRITY
-------------------------------------------------------------------------
7.1  Distribution: a self-contained folder (~290 MB unpacked, 97 MB
     zipped); no installation, no administrator rights, no writes
     outside its own folder other than user-directed exports.
7.2  Launcher integrity. The executable is the unmodified python.org
     CPython 3.13.14 windowed runtime carrying a valid Python Software
     Foundation Authenticode signature; the application starts through
     its embedded startup hook. This design eliminates the unsigned
     self-extracting stub that triggered antivirus false positives in
     earlier builds. The package was scanned clean by Microsoft Defender
     (definitions 1.455.81.0, 2026-07-11).
7.3  Pinned dependencies: numpy 2.4.4, scipy 1.17.1, matplotlib 3.10.8,
     Pillow 11.2.1, sv-ttk 2.6.1, cmcrameri 1.9, pywin32 311.
7.4  Diagnostics: launch failures write launcher_error.log beside the
     executable; a console interpreter (python.exe) is included for
     troubleshooting and does not start the application.
7.5  Quality assurance: 43 automated tests (parser, reduction, sessions,
     legend, engine provenance) plus a startup self-test; interface
     changes verified against screenshot probes in light and dark
     themes.

-------------------------------------------------------------------------
8. KNOWN LIMITATIONS
-------------------------------------------------------------------------
8.1  Windows 10/11 (64-bit) only; Windows 7 is not supported.
8.2  Preview status: this build is for evaluation and is not yet a
     tagged public release.
8.3  Drag-and-drop of folders onto the window is planned but not yet
     enabled.
